# 数控机床整机采购标讯分析技能

## 核心目标

从海量标讯中精准识别**数控机床整机采购项目**，排除备件、配件、维修、保养、改造、实训室建设等非整机采购。

---

## 完整工作流程

### 步骤1：搜索标讯

**工具**：知了标讯 CLI

```bash
# 搜索数控机床相关标讯
zlbx search-bids --keyword "数控机床" --category "A02050901" --days 30

# 搜索加工中心标讯
zlbx search-bids --keyword "加工中心" --days 30

# 搜索数控车床标讯
zlbx search-bids --keyword "数控车床" --days 30
```

**输出**：标讯列表JSON，包含 bid_id, title, money, province, caller_name 等字段

---

### 步骤2：初步相关度评估

**方法**：AI分析标题 + 关键词摘要（sm_names字段）

**评估规则**：

| 相关度 | 判定标准 | 后续处理 |
|:------:|----------|----------|
| 🟢 高相关 | 标题明确包含"数控机床/加工中心/数控车床"，无排除词 | 直接获取正文 |
| 🟡 中相关 | 可能相关，但需确认具体采购内容 | 获取正文深入分析 |
| 🔴 低相关 | 可能是配套设备或辅助采购 | 获取正文深入分析 |
| ⬜ 排除 | 标题/摘要包含排除关键词 | 跳过 |

**排除关键词清单**：
```
备件、配件、维修、保养、改造、实训室、实验室、建设、环评、
软件开发、系统集成、工业机器人（单独采购，非配套）、
传感器、刀具、刀架、电磁阀、伺服电机、驱动器
```

---

### 步骤3：同步到明道云（先查后写）

**工具**：明道云MCP API

**核心逻辑**：幂等同步（先查询是否存在，存在则更新，不存在则新增）

**工作表配置**：
- worksheet_id: `zlbx`

**字段映射**：
```javascript
const fieldMapping = {
  'bid_id':           '6a2d6c0c37004c0467cd3daf',
  'title':            '6a2d6c0c37004c0467cd3db0',
  'bid_type':         '6a2d6c9bae5739e8ccd1ec1c',
  'pub_time':         '6a2d6c9bae5739e8ccd1ec1e',
  'money_wan':        '6a2d6c9bae5739e8ccd1ec20',
  'province':         '6a2d6c9bae5739e8ccd1ec21',
  'city':             '6a2d6c9bae5739e8ccd1ec22',
  'caller_name':      '6a2d6c9bae5739e8ccd1ec24',
  'sm_names':         '6a2d6c9bae5739e8ccd1ec28',
};
```

**API调用（正确方式）**：

```javascript
// MCP连接配置
const mcpUrl = 'https://lcode.bj-fanuc.com.cn/mcp?HAP-Appkey=xxx';

// 1. 查询记录（精确匹配）
const result = await sendMcp({
  jsonrpc: '2.0',
  method: 'tools/call',
  params: {
    name: 'get_record_list',
    arguments: {
      worksheet_id: 'zlbx',
      filter: {
        type: 'group',
        logic: 'AND',
        children: [
          { type: 'condition', field: fieldMapping['bid_id'], operator: 'eq', value: String(bidId) }
        ]
      },
      pageSize: 1
    }
  }
});

// 2. 如存在则更新（rowId是驼峰格式）
await sendMcp({
  jsonrpc: '2.0',
  method: 'tools/call',
  params: {
    name: 'update_record',
    arguments: {
      worksheet_id: 'zlbx',
      row_id: record.rowId,
      fields: [{ id: fieldMapping['title'], value: '更新值' }],
      ai_description: `更新bid_id=${bidId}`
    }
  }
});

// 3. 如不存在则创建
await sendMcp({
  jsonrpc: '2.0',
  method: 'tools/call',
  params: {
    name: 'create_record',
    arguments: {
      worksheet_id: 'zlbx',
      fields: [...],
      triggerWorkflow: false,
      ai_description: `创建bid_id=${bidId}`
    }
  }
});
```

**⚠️ 常见错误**：
| 错误 | 原因 | 正确方式 |
|------|------|----------|
| `Tool not found: list_records` | 工具名错误 | 使用 `get_record_list` |
| 重复创建 | `contains` 模糊匹配 | 使用 `eq` 精确匹配 |
| `Map has no value for 'row_id'` | 参数名错误 | 使用 `row_id`（小写下划线）但值取 `rowId` |
| 更新失败 | 缺少 `ai_description` | 必须添加此参数 |

**幂等性验证**：
- ✅ 首次同步 → 新增
- ✅ 再次同步 → 更新（无重复）

**参考脚本**：`step3_upsert_mingdao.js`

### 步骤5：AI逐条精准判断

**方法**：逐条阅读正文内容，结合标题综合判断

**核心判断标准**：

#### ✅ 判定为【机床整机采购】的条件

1. **采购对象是完整机床设备**
   - 数控车床、数控铣床、加工中心
   - 明确标注型号（如6150、CK6132、VMC850）
   - 数量明确（如"采购数控车床22台"）

2. **采购内容描述为"设备采购"**
   - "采购数控机床1台"
   - "购置五轴加工中心设备"
   - 预算列入"设备购置费"

3. **采购目录匹配**
   - A02050901（金属切削机床）
   - A02050999（其他金属切削机床）

#### ❌ 判定为【非整机采购】的排除条件

| 类型 | 典型特征 | 排除理由 |
|------|----------|----------|
| **备件/配件采购** | 电磁阀、伺服电机、刀架、刀具、丝杠、导轨 | 机床零部件，非整机 |
| **维修/保养服务** | 设备维修、保养、检修、年检 | 服务类，非采购设备 |
| **改造/升级** | 设备改造、数控化改造、升级 | 现有设备改造，非新机采购 |
| **实训室建设** | 实训装置、实训室、产教融合、实训教学 | 综合性建设，含设备但非单纯整机采购 |
| **实验室建设** | 实验室、检测中心、研发中心 | 综合性建设项目 |
| **软件开发/系统** | 工业软件、MES系统、ERP、智能化系统 | 软件系统，非硬件 |
| **产业园/生产基地** | 产业园、产业化项目、生产基地 | 综合建设项目 |
| **环评/公示** | 环境影响评价、环境影响报告表 | 行政审批文件，非采购 |
| **资产处置** | 资产转让、资产变卖、报废处置 | 资产处置，非采购 |

## 存疑项目处理

**处理原则**：宁缺毋滥

当正文信息不足以明确判断时：
1. 标记为"⚠️ 存疑"
2. 记录具体疑点
3. 提请人工确认

**常见存疑场景**：
- 采购意向未明确具体设备清单
- 综合采购包含多种设备，需进一步拆分
- 标题与正文内容不一致

## 执行检查清单

- [ ] **步骤1**：使用正确关键词搜索，确保覆盖数控机床相关品类
- [ ] **步骤2**：逐一分析标题和摘要，避免遗漏重要信息
- [ ] **步骤3**：记录所有标讯到明道云，便于后续追踪
- [ ] **步骤4**：获取高/中相关度标讯的完整正文
- [ ] **步骤5**：
  - [ ] 逐条阅读正文（不能只靠标题）
  - [ ] 对照判断标准进行分类
  - [ ] 记录存疑项目待人工确认
- [ ] **步骤6**：将确认的整机采购项目更新到明道云

---

## 适用场景

- **新项目启动**：按此流程执行完整分析
- **增量数据处理**：步骤1使用时间范围筛选增量数据
- **结果复核**：对已有数据进行质量抽查

---

## 关键原则

1. **正文必须阅读**：不能仅凭标题判断，正文是最终依据
2. **宁缺毋滥**：存疑项目不贸然判定，提请人工确认
3. **记录完整**：所有分析结果记录到明道云，便于追溯
4. **持续迭代**：发现新类型及时更新排除关键词清单
