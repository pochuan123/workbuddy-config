# HAP 视图插件开发技能

## 简介

这是一个专业的技能，用于创建和开发明道云 HAP 自定义视图插件。提供完整的插件开发工作流：项目初始化、依赖安装、启动调试、字段解析、API调用、构建发布等全流程支持。

## 适用场景

- ✅ 数据看板、BI 驾驶舱
- ✅ 订单/任务看板视图
- ✅ 日历、甘特图视图
- ✅ 地图视图
- ✅ 简单列表、卡片展示
- ✅ 自定义业务视图

## 核心能力

1. **完整的开发工作流**
   - 7步完整流程（从环境检查到发布上线）
   - 自动选择合适模板（JavaScript/React/React-Tailwind/Vue）
   - 静默执行，用户无需手动操作

2. **字段类型处理**
   - 正确处理单选字段 (type 9)
   - 处理多条关联字段 (type 29)
   - 完整的字段类型对照表

3. **API 集成**
   - mdye 内部 API（当前表操作）
   - HAP V3 公开接口（跨表、高级功能）
   - 透视表统计 API

4. **BI 驾驶舱设计**
   - 业务分析师视角设计
   - 完整的指标定义规范
   - 数据可视化最佳实践

5. **开发调试支持**
   - 完整的问题排查流程
   - 快速诊断脚本
   - 常见错误解决方案

## 关键要点

### 用户反馈规则（必须严格遵守）

**❌ 严禁输出的内容：**
- 调试地址
- "请复制到明道云..."之类的指引
- 项目结构说明
- 命令执行日志
- "启动完成"、"编译成功"等技术状态

**✅ 正确的做法：**
- 启动服务器后，保持完全静默
- 简单回复："已完成,请在明道云中查看效果。"
- 修改后回复："已修改"
- 发布后回复："已发布"

### 开发服务器管理

- ✅ 使用 `run_in_background: true` 启动开发服务器
- ✅ 服务器应该一直保持运行状态
- ❌ 永远不要主动关闭开发服务器
- ❌ 只有用户明确要求时才关闭

### 构建和发布流程

- ✅ 每次修改代码后，立即执行 `mdye build && mdye push`
- ✅ 主动完成发布，不需要用户额外操作
- ❌ 不要等用户提醒才发布

### 模板选择建议

- 📊 **数据看板、BI 驾驶舱** → `React` 或 `React-Tailwind`
- 📅 **日历、甘特图** → `React`
- 🗺️ **地图视图** → `React`
- 📝 **简单列表、卡片展示** → `JavaScript` 或 `React-Tailwind`

### 字段类型重要提示

- ⚠️ **Type 9** = 单选 (SingleSelect) - 注意不是 type 11
- ⚠️ **多条关联字段** (type 29) 返回数字，需要调用 `getRowRelationRows` API
- ⚠️ **单条关联字段** (type 29) 返回 JSON 字符串，可以直接解析
